# Blurr AI Assistant - Business Model V2.0

## üéØ Core Principle: No Limits + Value-Added Premium

**Philosophy:** Users already pay for their AI compute (BYOK). We should never charge them again for the same usage. Instead, we monetize by providing **premium value-added features** that are expensive to maintain or deliver massive ongoing value.

---

## 1. Business Model Summary

### The Problem with the Old Model

**Old approach (REJECTED):**
- Free: 10-20 agent runs/month
- Pro: Unlimited runs

**Why this was wrong:**
- **Double-dipping**: Users pay for API keys, then we limit their usage
- **Creates resentment**: "I'm already paying OpenRouter, why are you restricting me?"
- **Hurts virality**: Can't demo freely, limited word-of-mouth
- **Misaligned incentives**: We profit from restricting access

### The New Model: Unlimited Core + Premium Superpowers

**Core principle:** Everyone gets **unlimited everything** in the free tier for all basic functionality. We only charge for premium features that:
1. Cost us significant money to provide (web search APIs, server hosting)
2. Require ongoing maintenance and infrastructure (workflow scheduling, team features)
3. Deliver massive value that justifies premium pricing (local models, advanced analytics)

**Result:**
- **Fair**: You control your AI costs completely
- **Viral**: Unlimited free demos and word-of-mouth
- **Aligned**: We succeed by adding real value, not restricting access
- **Premium**: Pro users get genuine superpowers worth paying for

```kotlin
enum class PlanFeature {
    // Always free (no flag needed)
    UNLIMITED_AGENT_RUNS,
    UNLIMITED_CHAT,
    BYOK_SUPPORT,
    PHONE_AUTOMATION,
    BASIC_TOOLS,
    CONTEXT_PRESERVATION,
    
    // Pro features
    WEB_SEARCH,
    DEEP_RESEARCH,
    FULL_WORKFLOW_BUILDER,
    WORKFLOW_SCHEDULING,
    ADVANCED_MULTIMODAL,
    LOCAL_MODELS,
    OFFLINE_MODE,
    AGENT_TEMPLATES,
    WORKFLOW_MARKETPLACE,
    FULL_GOOGLE_WORKSPACE,
    STORAGE_100GB,
    PRIORITY_SUPPORT,
    
    // God Mode features
    TEAM_SHARING,
    COLLABORATIVE_EDITING,
    PRIVATE_MCP_HOSTING,
    WHITE_LABEL,
    CUSTOM_DOMAIN,
    SSO_INTEGRATION,
    ADVANCED_ANALYTICS,
    AUDIT_LOGS,
    RBAC,
    DEDICATED_SUPPORT
}

object FeatureGate {
    fun hasAccess(feature: PlanFeature, userPlan: Plan): Boolean {
        return when (feature) {
            // Free tier - always true
            PlanFeature.UNLIMITED_AGENT_RUNS,
            PlanFeature.UNLIMITED_CHAT,
            PlanFeature.BYOK_SUPPORT,
            PlanFeature.PHONE_AUTOMATION,
            PlanFeature.BASIC_TOOLS,
            PlanFeature.CONTEXT_PRESERVATION -> true
            
            // Pro features
            PlanFeature.WEB_SEARCH,
            PlanFeature.DEEP_RESEARCH,
            PlanFeature.FULL_WORKFLOW_BUILDER,
            PlanFeature.WORKFLOW_SCHEDULING,
            PlanFeature.ADVANCED_MULTIMODAL,
            PlanFeature.LOCAL_MODELS,
            PlanFeature.OFFLINE_MODE,
            PlanFeature.AGENT_TEMPLATES,
            PlanFeature.WORKFLOW_MARKETPLACE,
            PlanFeature.FULL_GOOGLE_WORKSPACE,
            PlanFeature.STORAGE_100GB,
            PlanFeature.PRIORITY_SUPPORT -> 
                userPlan == Plan.PRO || userPlan == Plan.GOD_MODE
            
            // God Mode features
            PlanFeature.TEAM_SHARING,
            PlanFeature.COLLABORATIVE_EDITING,
            PlanFeature.PRIVATE_MCP_HOSTING,
            PlanFeature.WHITE_LABEL,
            PlanFeature.CUSTOM_DOMAIN,
            PlanFeature.SSO_INTEGRATION,
            PlanFeature.ADVANCED_ANALYTICS,
            PlanFeague.AUDIT_LOGS,
            PlanFeature.RBAC,
            PlanFeature.DEDICATED_SUPPORT -> 
                userPlan == Plan.GOD_MODE
        }
    }
}

enum class Plan {
    FREE,
    PRO,
    GOD_MODE,
    PRO_TRIAL  // Special state for trial users
}
```

### 5.2 Trial Implementation

**Trial Logic:**
```kotlin
data class UserSubscription(
    val plan: Plan,
    val trialStartDate: String?,  // ISO-8601
    val trialEndDate: String?,    // ISO-8601
    val subscriptionStartDate: String?,
    val subscriptionEndDate: String?,  // null if active
    val isTrialActive: Boolean = false
)

object TrialManager {
    const val TRIAL_DURATION_DAYS = 14
    
    fun startTrial(userId: String): UserSubscription {
        val now = Instant.now()
        val endDate = now.plus(TRIAL_DURATION_DAYS, ChronoUnit.DAYS)
        
        return UserSubscription(
            plan = Plan.PRO_TRIAL,
            trialStartDate = now.toString(),
            trialEndDate = endDate.toString(),
            subscriptionStartDate = null,
            subscriptionEndDate = null,
            isTrialActive = true
        )
    }
    
    fun checkTrialStatus(subscription: UserSubscription): TrialStatus {
        if (!subscription.isTrialActive) return TrialStatus.NOT_IN_TRIAL
        
        val endDate = Instant.parse(subscription.trialEndDate!!)
        val now = Instant.now()
        val daysRemaining = ChronoUnit.DAYS.between(now, endDate)
        
        return when {
            now.isAfter(endDate) -> TrialStatus.EXPIRED
            daysRemaining <= 3 -> TrialStatus.ENDING_SOON(daysRemaining.toInt())
            else -> TrialStatus.ACTIVE(daysRemaining.toInt())
        }
    }
    
    fun convertToSubscription(userId: String, plan: Plan): UserSubscription {
        val now = Instant.now()
        
        return UserSubscription(
            plan = plan,
            trialStartDate = null,
            trialEndDate = null,
            subscriptionStartDate = now.toString(),
            subscriptionEndDate = null,  // Active subscription
            isTrialActive = false
        )
    }
}

sealed class TrialStatus {
    object NOT_IN_TRIAL : TrialStatus()
    object EXPIRED : TrialStatus()
    data class ACTIVE(val daysRemaining: Int) : TrialStatus()
    data class ENDING_SOON(val daysRemaining: Int) : TrialStatus()
}
```

## 9. FAQs

**Q: Why is the free tier unlimited?**
A: You already pay for AI compute through your BYOK API keys. Charging you again for usage would be double-dipping. We only charge for premium features that cost us money to provide or deliver massive value.

**Q: What if I don't have an API key?**
A: We'll guide you to get one! Many providers offer free tiers (OpenRouter, Groq) or generous credits (AIMLAPI gives $10 free). Setup takes 2 minutes.

**Q: Can I switch between providers?**
A: Absolutely! Configure multiple providers and switch anytime. Your keys are encrypted and stored locally on your device.

A: Everything in Pro - web search, workflows, local models, advanced generation, 100 GB storage. Full access, no credit card required to start.

**Q: Can I cancel Pro anytime?**
A: Yes, cancel anytime. You'll keep Pro until the end of your billing period, then automatically switch to the unlimited free tier.

**Q: Why should I pay for Pro if free is unlimited?**
A: Pro unlocks features that require ongoing costs (web search APIs) or significant infrastructure (workflow scheduling, MCP hosting). These features deliver massive value beyond basic AI chat.

**Q: Do you sell my data?**
A: Never. Your API keys are encrypted locally. We never see them. Our business model is transparent: free unlimited core + premium value-added features.

**Q: What's the difference between Pro and God Mode?**
A: God Mode adds team features (5 users, shared workflows), enterprise capabilities (private MCP hosting, SSO, white-label), and dedicated support.
